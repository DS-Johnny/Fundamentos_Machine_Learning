# Cria uma instncia de um classificador de rvore de deciso. A profundidade mxima limita o nmero de nveis da rvore, ajudando a controlar o overfitting.
model = tr.DecisionTreeClassifier(max_depth=38)

# Treina o modelo de rvore de deciso usando os dados de treinamento x (caractersticas) e y (etiquetas).
model.fit( x,y )

# Usa o modelo treinado para prever as etiquetas dos dados de treinamento x. As previses so armazenadas em yhat.
yhat = model.predict(x)

# Calcula a acurcia do modelo comparando as etiquetas verdadeiras y com as previses yhat. A acurcia  a proporo de previses corretas feitas pelo modelo.
acc = mt.accuracy_score( y, yhat)
print(f'Accuracy Over Training: {acc}')

Accuracy Over Training: 1.0

Quando a sada Accuracy Over Training: 1.0, isso significa que o modelo de rvore de deciso acertou todas as previses nos dados de treinamento. Aqui esto algumas consideraes e possveis implicaes desse resultado:

Possveis Implicaes: Overfitting:

Definio: Overfitting ocorre quando o modelo se ajusta muito bem aos dados de treinamento, incluindo o rudo e as caractersticas especficas do conjunto de dados de treinamento, mas no generaliza bem para novos dados. Sinal de Overfitting: Uma acurcia de 1.0 (ou 100%) no conjunto de treinamento um forte indicativo de overfitting, especialmente quando a profundidade mxima da rvore (max_depth=38) alta. rvores de deciso profundas podem memorizar os dados de treinamento. Complexidade do Modelo:

Profundidade da rvore: A profundidade mxima de 38 bastante grande, o que permite rvore capturar muitos detalhes dos dados de treinamento. Isso pode levar ao overfitting. Simplificao: Voc pode tentar reduzir a profundidade mxima da rvore para ver se isso melhora a generalizao do modelo nos dados de teste.

# Separa os dados em Treinamento e Teste
x_train, x_test, y_train, y_test = ms.train_test_split(x, y, test_size=0.2, random_state=random_state)

Separao dos Dados:

  • x_train, x_test, y_train, y_test = ms.train_test_split(x, y, test_size=0.2, random_state=random_state):
  • x_train e y_train so os dados de caractersticas e etiquetas que sero usados para treinar o modelo.
  • x_test e y_test so os dados de caractersticas e etiquetas que sero usados para testar o modelo.
  • test_size=0.2 especifica que 20% dos dados sero reservados para o conjunto de teste, enquanto 80% sero usados para treinamento.
  • random_state=random_state garante que a diviso dos dados seja reproduzvel, ou seja, voc obter a mesma diviso dos dados se executar o cdigo novamente com a mesma semente aleatria.

# Definindo uma lista de profundidades para testar
values = [i for i in range(1,60)]

# Inicializando uma lista para armazenar as acurcias de teste para diferentes profundidades
test_score = []

# Iterando sobre cada valor de profundidade
for i in values:
    # Modelo treinado com os dados de treinamento, para cada valor de profundidade
    model = tr.DecisionTreeClassifier( max_depth=i)
    model.fit(x_train, y_train )

    #    # Classificando os dados de teste (x_test) usando o modelo treinado com o valor atual de profundidade
    yhat_test = model.predict( x_test )
    acc_test = mt.accuracy_score(y_test, yhat_test)  # Calculando a acurcia do modelo nos dados de teste

    test_score.append(acc_test)

Anlise do Resultado

Aps executar esse cdigo, voc ter uma lista (test_score) que contm a acurcia de teste para cada profundidade da rvore de deciso de 1 a 59. Isso permite que voc visualize como a profundidade da rvore afeta o desempenho do modelo.

Voc pode usar a lista test_score para plotar um grfico e visualizar o desempenho do modelo em funo da profundidade. Isso pode ajudar a identificar a profundidade tima da rvore que oferece o melhor equilbrio entre ajuste e generalizao. Aqui est um exemplo de como voc pode fazer isso:
